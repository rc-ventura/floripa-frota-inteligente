### External QA Opinion (Devin)
Landed after the initial report (static-analysis mode — exec blocked in its session). **Verdict: APPROVED with 2 MEDIUMs, both non-blocking test-hardening:**
- Coalesce semantics confirmed identical in SQLite and Postgres (NULL→`''`, enforced collision); model and migration consistent; mirrors `ux_alerta_ativo`.
- Batch-rebuild regression risk assessed LOW; CHECK/FK indirectly guarded by existing tests; the two `ix_` indexes not asserted in-suite (QA covered this empirically via `.schema` + round-trip).
- MEDIUM 1: no in-suite introspection assertion that `ux_multa_upsert`/`ix_multa_placa_data` exist → **applied post-review**: `test_criacao_do_zero` now checks the catalog directly (`sqlite_master`/`pg_indexes` — the SQLite inspector omits expression indexes, same R6 limitation). Suite: 17 passed.
- MEDIUM 2: upgrade of a **populated** 0001 DB already containing NULL-condutor duplicates would fail at `create_index` (the new unique index rejects pre-existing dups) — arguably correct (surfaces dirty data), untested. Accepted for the PoC: 0001 never shipped with data; noted for any future data-bearing upgrade.
- LOW (suppressed as by-design): `condutor_pseudo=''` would collide with NULL via coalesce — intended sentinel semantics per ADR-004; domain is NULL or `COND-NNN`.
